# taskloop 契约卡(始终加载;新系统方言)

默认轻量:能直接做的直接做。微小改动(单文件、无数据/接口影响)不开环,
一句"现状 → 期望"即可。只有用户显式要求、机制未收敛,或不可逆/高风险
改动才开任务环。

- 开环 → `taskloop open`:goal + **出生即红**的 criterion + **alignment**
  (绿为什么⇒目标;不覆盖什么)+ envelope。三者缺一先索要再动手。
- 循环体 → 改 → 证据 → 停(闸门现场跑判据裁决)。轮间不问人;判据红的
  反馈尾就是下一轮的输入。
- 人持有四个动词:**拍板**(方案/预算)、**代行**(部署/凭证等机器够不着
  的外部动作)、**验收**(终局)、**转向**(中途改向是一等输入,立即吸收
  并继续,不视为打断)。循环开口要人当时钟/判停器/守卫/记忆 → 是缺陷,
  记入元层。
- 收口只有四条路:判据绿(机器)、`not-needed --evidence`、
  `abandon --reason`、挂起(`suspend --judgment`,任务保持 open)。
  没有 claim-based 成功。
- 任何"已完成/已验证"声明必须指向本会话的真实工具输出;工具失败如实报告。
- 删除/清空/git 写操作 → 仅在用户明确要求且 envelope 授权后进行。
