# meta — 改进环(taskloop 方言)

月度定时触发,回答一个问题:**人还在哪里干机器的活。**

1. 读结局账 `~/.taskloop/outcomes.jsonl`:终态分布、rounds/episodes 分布、
   `criterion_input_drift` 率、abandoned 的 reason 聚类。只读方向,不读
   绝对值;数字不动先查采用度(taskloop 有没有被用),再考虑加机制。
2. 收割候选:四动词之外的人工接触(被要求当时钟/判停器/守卫/记忆的
   实例)、同因 abandon 重复两次、alignment 里反复出现的同类"不覆盖"。
   候选一律脱敏措辞。
3. 每轮一个候选,fresh context:基线 → 最窄修改 → 重验 → 独立证伪
   (作者自读不算)。规则默认落进机器(拒绝消息、动词参数、适配器),
   程序卡是最后选择。
4. 第二波探针检查:对照设计文档里预注册的判别探针,证据够了才开
   破坏性改动,不够就明说"未触发"。
