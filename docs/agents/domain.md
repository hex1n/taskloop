# Domain Docs

This repository uses a single domain context.

## Before exploring

- Read root `CONTEXT.md` when it exists.
- Read system-wide ADRs under `docs/adr/` when they exist and touch the work.
- If either location is absent, proceed without inventing a replacement.

## Vocabulary

Use the domain terms defined by `CONTEXT.md`. Until a glossary exists, preserve the established vocabulary in `AGENTS.md`, the runtime contracts, and accepted plans; do not replace terms such as authority, attachment, task, session, criterion, receipt, projection, placement, or host approval with near-synonyms.

## ADR conflicts

Surface any conflict with an existing ADR explicitly. Do not silently override it in a spec or ticket.
