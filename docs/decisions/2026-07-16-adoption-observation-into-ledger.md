## 票面与机械事实

票面四裁决项（记什么／记成什么／与 gate 的关系／degrade-open 的分寸）。开工核实的机械事实，其中两条票面没有：

1. **传感器只有一个**：PreToolUse hook（`lib/application.mjs:899/911`）。hook 今天未装（owner 明确选择）→ 观测源缺席。
2. **存储是瞬态的**：`.taskloop/untracked-writes.json`，session 分桶、24h TTL、写失败静默（「the nudge must never break the tool call」）。
3. **三条销毁路径，其中一条票面没写**：`open` 后 `clearUntracked`（`application.mjs:490`，**抹全部 session 不只开单者**）；TTL；以及**毒药剪枝**（`untracked.mjs:58-59`）——磁盘上没有对应文件的条目按误归因剪掉，而**被 deny 的写入恰好不落盘 → 被 deny 的目标在下次观测时被当毒药剪掉**。今天的状态文件结构上装不下「被拒绝」这个最重要的物种。
4. **deny/notice 这个决定本身无处记录**——状态文件只存文件列表，不存闸门开了什么火。
5. **事件流结构上 task-scoped**：`event-store.mjs:335` 强制每条事件 `task_id`（UUID）+ 任务内序号 ≥1。「非采用」没有 task 可挂——票面「挂 `task_opened` 更省」实为「现行框架只装得下挂 task 的形态」。

## 决议

### 1. 消费者问题 = 物种学（B 为主，A 只作副产品）

meta-loop 拿这份观测回答的是「**绕过的是什么活、在哪一步绕的、后来补单没有**」——给雾区「无机器判据的仓内工作」供有形状的样本，不是采用率仪表盘。采用率（A）的分母**结构上永远测不准**（传感器只看得见 hook 在场时的写入），拿有洞的分母算率是生产假精度——16 决议 3「视野外静默退化成裸声明」的应用。A 只作副产品，不承诺精度。

传感器（在场时）分得开的物种：S1 nudge 后终未开单（真非采用）／S2 nudge 后补单／S3 第 2 文件被 deny／S4 开单期间 foreign session 的 untracked 写入／S3′ deny 后 owner 卸 hook 绕过（07-15 现场，只能以「边界消失」形式观测——归决议 4）。

### 2. 记什么：事实集 + 认识论标注 + 隐私治在投影层

- **事实集**：`acting_session`（hook 在场时是观测，票 12 已裁）、时刻、`targets_parsed`（repo-relative）、闸门开火种类（notice/deny）、`foreign` 标志、`during_task`（观测时刻开着哪张单——当时是本地事实，传感器手上就有，照 16 决议 2 记录）。
- **两样明确不记**：disposition（「随后补单没有」= fold 按 session join `task_opened`，读时投影——票 11 的教训，别把投影快照进事件）；累计计数（可导出）。
- **认识论标注**：PreToolUse 里的写入目标是**从工具调用解析出来的猜测**，不是落盘观测——毒药剪枝的存在本身就是它会错的证据。字段如实叫 `targets_parsed`，fold 读时想核对落盘自己核。「申请时刻的本地物种是声明」（16 决议 2）在传感器层的应用：别把解析猜测打扮成观测。
- **隐私治在投影层，不治在传感器层**：路径住 repo-local 证据书；跨仓全局投影只带计数/形状聚合（投影是读时选择）。反过来（为隐私不记路径）等于为副产品 A 自宫主产品 B——没有路径，样本就没有形状。
- **事件量的界**：瞬态状态文件继续当 session 内去重游标，只在「新文件首次入桶」或「notice→deny 升级」时发记录——量 ≈ 不同文件数 + deny 次数。

### 3. 记成什么：独立证据书，task 流一字不动

第一性区分：**状态溯源事件**（被 replay 出状态，序号/事务/cursor 纪律全为「重放得到唯一正确状态」服务）与**证据记录**（无状态机、无不变量、永不被 replay，唯一用途是被读者读）。非采用观测是后者——把证据塞进状态溯源流，等于给最脆弱的代码路径（近三个 commit 全在修 replay/cursor）加一个永久的「跳过非状态记录」分支。

- **形态**：`.taskloop/` 下独立追加流（JSONL），记录 `{seq, at, acting_session, foreign, gate, targets_parsed, during_task}`。命名按「记观测不记解读」——叫 untracked 观测，不叫 adoption（adoption 是读者的解读词）。
- **完整性靠序号不靠框架**：per-session 单调序号（计数器住现有瞬态状态文件）。写失败 → 序号跳空 → 读时可见；状态文件丢失 → 序号重置 → 读者见重复序号 = 「传感器状态被重置过」。**两种失败都朝诚实方向退化**（只会谎报「可能丢了」，不会谎报「什么都没丢」）。
- **「一本账」是 fold 的输出，不是一个文件**：09 决议 3/9 把 fold 与完整性判定归了 `ledger --json`——物理两本书（状态书 + 证据书）、逻辑一本账，恰是那两条决议为之而生的形状。
- **勘误（本 session 自纠）**：grilling 中我先以「耐久性纪律不同」论证分流，**部分是错的**——hook 路径今天就在做事务提交（`application.mjs:921-927`）。真正的结构障碍是聚合身份，不是耐久性。
- 挂 `task_opened` 的选项正式处决：只有 S2 会被记，S1/S3 永不入账——票面自己写的幸存者偏差原样复现。

### 4. 与 gate 的关系：模式梯，闸门可降档、观测不断流

**传感器就是 hook 本身**——hook 被卸，进程不跑，「hook 移除后记账仍发生」物理上不存在。解耦的唯一真实形态：**把闸门从 hook 里降档拆出来，不是把记账搬出 hook**。

- **模式梯**：hook 支持 `{observe, nudge, deny}` 三档。**任何档位都写证据书**，记实际发生的（当时档位 + 实际动作），不记反事实。
- **07-15 卸载的病理重述**：deny 咬了没有合法形态的活——那是 **deny 档的病，不是传感器的病**；全有全无开关把两者焊死，卸 deny 只能连传感器一起卸。
- **立即红利**：地图「PreToolUse 等雾区裁完再装」封印的是 deny 档；observe/nudge 档不咬人，**传感器不等雾区**。「原样装回被再卸」的预言对降档 hook 不成立。**owner 裁：装回停 nudge 档**（observe 拿不到「被提示后仍未开单」这层物种信息）。deny 档继续等雾区「无机器判据的仓内工作」。
- **卸载自身不可自观测——本体论限制照实记**：检测器测不到自己的缺席。处置照 16 决议 3：经 `taskloop hooks` 动词的装/卸/换档写过渡记录进证据书；fold 读时查当前 settings 报**当前**武装状态；过渡记录覆盖不到的区间报 **unknown 不报 zero**——绕开动词直改 settings.json 的卸载，承认看不见。
- **S3′ 获得户口**：「owner 降档/卸载」经动词走时本身成为证据书一行——最该被记的那次非采用，在新形态下恰好是最先被记的物种。

### 5. degrade-open 的分寸：写侧铁律保留，fail-loudly 住读侧

- **写侧阻断被否，理由是 S3′ 教训一般化**：观测仪器的代价一旦高过被观测的东西，仪器就会被拆掉，然后什么都观测不到。让 telemetry 写失败阻断用户工具调用 = 亲手制造下一次卸载——fail-loudly 在写侧自败。
- **读侧完整性成为 fold 的一等输出**（09 决议 9 延伸到证据书）：`ledger --json` 报四样——当前武装状态+档位、已知过渡记录、per-session 序号跳空/重置发现、**每个查询窗口的覆盖判定 covered/gapped/unknown**。含糊计价，永不沉默——「没有行」永远伴随覆盖判定。
- **Stop census 补强**（便宜，机制已在场）：Stop 闸门已装回、每次 session 停止都在跑——顺手写一行 `{at, session, pretooluse_armed, mode}` 进证据书。「零行」从 unknown 变成「窗口内每个 session 停止时传感器都在场」的逐 session 见证。

## 根据

**闸门是策略，观测是事实；策略可以降档、可以卸载，事实一旦发生就该入账。**同形家族的收束：06 手上有 `output_tail` 却扔了／07 有 tail 没拿它分类／08 有 `acting_session` 没拿它判独立性／13 有 untracked 观测却当 nudge 用完即弃——四例同根，修法都是**把观测从执行路径的副产品升格为账本的一等公民**。

## 执行项（寄存 #04）

1. **证据书建设**（决议 2/3）——独立追加流 + 事实集 + per-session 序号；task 事件流框架零改动。
2. **模式梯 + 传感器复活**（决议 4）——三档 hook，全档位记录；落地后装回 PreToolUse 停 nudge 档。**排序提示：它同时解锁票 12 `agent_id` 注入的「依赖：装 PreToolUse」**——那条依赖从雾区解锁出来，不必等「无机器判据的仓内工作」裁决。
3. **fold 扩展**（决议 3/5）——`ledger --json` 合并证据书 + 完整性四样输出。
4. **过渡记录**（决议 4）——`taskloop hooks` 装/卸/换档写过渡行。
5. **Stop census**（决议 5，可选补强）——session 停止时写传感器在场见证。

## 限制与遗留

- 传感器缺席在过渡记录覆盖外**永远不可证**——只能 unknown 计价，这是本体论限制不是缺陷。
- nudge 档对无合法形态的活仍是底噪（stderr 一行）——那条雾（「无机器判据的仓内工作」）本票不解决，只给它装上传感器：它等的「同类样本」将以 untracked 观测行的形式积累。
- 证据书 schema 细节（文件名、字段版本化）是 #04 的 spec 工作。
