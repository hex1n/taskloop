- 收口：owner 逐项确认（两本账都记／不可逆硬边界 认／change_classes 并轨 认／梯顶选项 1）

## 票面与重写

票面把 `machineRiskFloor` 的 6 个输入按「撒谎的代价」排开，得出「6 个里 4 个是声明的函数」，并问三个物种（声明／自缚声明／观测）的博弈等价性。**这个问题问错了方向**——按本图校准提示的原话，它拿现有输入定义了问题（本 session 第一轮照样栽进去，被 owner 当场纠正——本图第七次「拿当前实现当参照系」）。先问「门该由什么构成」，答案变成：

评审存在的第一性理由是**判据是真相的有损代理**（票 14 的「影子」、票 10 的「红→绿自证」）。门要估两个量：**缺口**（判据离真相多远——谁写的、盖住多少、中途改没改）与**赌注**（真错了摊多大）。评审值 = 缺口 × 赌注。

owner 裁定，taskloop 的评审**两本账都记**：

- **账 A（补缺口）**：判据够不到的地方需要第二双眼——「任务完成了，另一个来审查还有哪些遗漏、哪些问题」。
- **账 B（权力钥匙）**：有些权力不论把握多大都要第二方点头——被质疑的不是能力，是单钥匙。

## 主结构裁决：两本账住不同时刻

- **账 B 的钥匙必须在权力开火之前转动**——事后才转的钥匙不叫钥匙，叫事故调查。家在**发权时刻**（open/amend）。
- **账 A 必须在活干完之后**——没干完的活谈不上遗漏。家在**关单时刻**。

现行地板把两本账塞进关单一个时刻结算，只能用 `max()` 焊成一个标量——**标量是症状，单一时刻才是病**。具体开火：今天自批 destructive 的路径（自发权 → 地板抬 critical → 关单时 second_model 评审）是**拿账 A 的货币（事后评审）买账 B 的商品（权力放行）**——07 命名的「付费通行」由此看清成色：第二把钥匙在导弹落地之后才转。

账本佐证（v2 fold，10 个终态任务）：**10/10 effective ≥ substantial，`routine` 零出现**——任何先开火的输入都把 risk 抬离 routine（缺口轴 10/10 开火：07 的饱和链），于是赌注轴上两条 `risk === "routine"` 守卫的规则从未被求值。**11 的死规则（`many_touched_files` 0/10）不是自己死的，是被 `max()` 焊接掐死的**；07 的饱和链、11 的死规则、16 的「4 个声明」是同一个结构错误的三个症状。

## 决议

### 1. 账 B 硬边界 = 不可逆权力

按可逆性划线：**可逆**权力（仓内写、git 可回滚的操作）的事后评审 + 回滚权是**功能性钥匙**——发现错了还能撤，等效事前；**不可逆**权力（publish、仓外删除、push 到共享远端）撤不了，事后的一切只是审计。故：

- 不可逆权力的自批**不再用「抬地板买事后评审」结算**；处置 = 记账 + 依赖宿主钥匙／召唤人。
- `granted_by` 上**不立闸门**：runtime 验证不了「人」（`--granted-by user` 同样是 agent 敲的，`lib/application.mjs:131-133`）；焊死诚实的门只生产洗过的门——08/12 先例第三次成立，记账优于闸门。
- 真正**不可伪造**的事前钥匙在**宿主层**（host 权限提示；08 的 `/sandbox` = 预先整体转开）。taskloop 的账 B 工作是**记录宿主钥匙状态**——可观测性探测出**票 17**（host-neutral 判据把关：只在半个宿主成立的锚不进内核）。
- 哪些 grant kind 落不可逆侧是 #04 的 spec 工作（publish 确定在内；destructive／git 按作用域与远端共享性逐一按「撤得回吗」归类）。

### 2. 关单地板 = 纯账 A = 缺口 × 实得赌注

**每个时刻读它的本地物种：申请时刻的本地物种是声明（申请本来就是「我想要」），结算时刻的本地物种是观测（事实只在事后存在）。** 现行地板的病一句话：**在结算时刻读预报**——读 `grants` 与 envelope 宽度（开单时的申请），而 touched files、实际写入根、检测器见过的命令形状在关单时刻全部存在（v3 fold 即得；接受 v3 依赖，同 11 决议 3 的取舍，owner 已裁）。

- 缺口因子：provenance／coverage／时序事件（`risk_floor_events`，`application.mjs:790`——真观测）照旧，这半边健康。
- 赌注因子：改读**实得**。
- 「6 输入 4 声明」就此溶解：不是给声明找更可信的替身，是各回各的时刻。「申请了权没用」不再被冤枉顶级；「没申请却干了大事」的沉默者第一次被照到——**沉默不再省钱**，即票面「奖励沉默、向诚实的人收税」的解，且不需要逼任何人诚实。

### 3. 检测器边界有声化

「自缚声明」这个物种不成立：其绑定力是**检测器的属性，不是声明自己的**。`commandSafetyFailure` 自述 "it raises the cost of the obvious dangerous forms, it is not a sandbox"（`lib/supervision.mjs:129-134`）；destructive 检测是四条正则（`:156-159`），`node -e 'fs.rmSync(p,{recursive:true})'` 一条都不匹配。视野内 ≈ 观测，视野外**静默退化成裸声明**——比裸声明更坏一层：裸得看不见。

处置：**盲区自己可观测**——hook 解析不出写入目标时记 `["<command>"]`（`lib/application.mjs:950`），「我读不出你写了哪」这件事 hook 当场知道。**把「写入不可归因」升格为缺口信号**：看得清的按实得算赌注，看不清的按「看不清」抬缺口。检测器边界内外的切换从静默退化变成有声升级，**含糊本身计价**，谎不需要被识破。

### 4. 声明的统一处置：永不作闸门键，永远记录在锚旁

- **`change_classes` 废除独立法律效力，并轨 `declared_risk`**（同一物种：自愿、只抬的自缚声明；词表 schema/public_contract/… 降为结构化 reason 保留）。它从没解过沉默的说谎者——只捕获过诚实人：账本唯一一次声明（16da9ae1 `public_contract`）是在已被 grants 顶到 critical 之后作出的，**非边际**——并轨零损失。语义级赌注对沉默者维持现状的盲；path→class 映射是仓库特定知识，按 09 的内核判据**住消费者层**，不进内核。
- **`level` 与 `granted_by` 同判**：词照旧过门（需求侧由风险推导，健康），满足侧永不单独作数——评审事件并排记录宿主递来的不可伪造锚（`agent_id` 在否、`acting_session` 适配器状态）。**「声称 fresh_context 却无锚」成为账本里可查询的类**，meta-loop 抽样审计的现成靶子。谎不被拦，但留指纹。（12 的 P1→P2 洗白论证在此复用：闸门让诚实的自敲灭绝、洗过的图章免费。）

### 5. 评审门终点不是 judgmentloop；梯顶按「接受」处置

- 内核**够不到**「人的显式验收」：owner 敲的 `review` 与 agent 敲的在 CLI 层逐字节不可分（本体论边界；08 的 D2「不可强制」同源）。内核天花板 = **必要条件机器强制 + 出处全记录 + 诚意抽样审计**；人的验收真实存在，但住 skill 层（judgmentloop 编排），内核只记录它的声称。P2/P3 残差不是修掉的，是**定价后接受的**。
- 梯顶：`second_model` 的模型多样性在 Claude 的 hook payload **不可观测**（无 model 字段；Codex 有——不对称，内核押它即 host-dependent）。今天在 Claude 上，顶档比第二档多出来的只是一个词。owner 裁**选项 1：接受**——梯子照旧，模型声称记账不验证，等雾区毕业条件（Claude payload 出现模型标识）再补锚；不加「人」档（同样不可验证，徒增一个可敲的词），不降顶。

## 证据

- `machineRiskFloor`：`lib/task-engine.mjs:112-133`；两条 routine 守卫 `:130-131`。
- `acceptedReview` `:135-143` 不看 reviewer；`REVIEW_ORDER` `:78` 无 self_reread（结构排除正确——账本 99ef2c2a 真敲过 `self_reread/claude-opus`，记了账、没过门，行为正确）。

- 检测器自述 `lib/supervision.mjs:129-134`；每命令安全检查 `lib/application.mjs:932-935`（在 write-shaped 短路之前——**runtime 手上有命令形状，地板读的却是 grants**：同形「手上有观测却读声明」，本票定性为时刻错位而非又一次字段遗弃）。
- v2 账本 fold：10/10 effective ≥ substantial；routine 零出现。
