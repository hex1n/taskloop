## Blocking

### B1 — rooted Windows paths (single leading backslash) still evaded the guard, and a test had frozen that gap as intended behavior

`retired supervision runtime` — `WINDOWS_ABSOLUTE_PATH`, consumed by `readShellValue` and `resolveShellTarget`

Windows has a fourth absolute form neither earlier round enumerated: a **rooted** path — one leading backslash, no drive letter, e.g. `\Users\hexin\Desktop\workloop\.git\config` — which resolves against the root of the current drive. `path.win32.isAbsolute` reports `true` for it, and for an agent working in-repo on that drive it names byte-identically the same file as the drive-letter spelling. The regex accepted only `letter:` or exactly two leading backslashes, so a single leading backslash fell through to POSIX escaping and was mangled. Reviewer reproduced:

```
echo hi > \Users\hexin\Desktop\workloop\.git\config
  parsed target -> "UsershexinDesktopworkloop.gitconfig"
  controlPlaneWriteFailure(...) -> null        // NOT denied
echo hi > C:\Users\hexin\Desktop\workloop\.git\config    (same file, drive form)
  controlPlaneWriteFailure(...) -> denied
```

**The test-quality half**: the round-two test at `tests/foreign-session-scope.test.mjs` did not merely omit this shape, it asserted the bug as correct —

```js
assert.deepEqual(targetOf(String.raw`echo x > \server\share`), ["servershare"]);
// "A single leading backslash is NOT a Windows root, so POSIX escaping still applies"
```

That premise is false. This is the same defect class the first round criticized (a test that engineers away the precondition it claims to cover), committed while fixing it — and it is very likely why rounds one and two both cleared "standard Windows shapes" as correct.

**Resolution**: replaced the regex with a `windowsRootedPath()` predicate covering all four shapes — drive root, drive-relative, UNC, and rooted. The rooted branch requires a later separator so a POSIX escape of a command word (`\ls` defeating an alias) is not misread as a path; misreading a command word would break command classification, which fails toward allow. The false assertion was rewritten to assert the rooted path survives, plus two new negative assertions (`\ls` stays a command word, `sed s:/old\ a:/new:` keeps normal escape processing). Added "every Windows root shape reaches the control-state guard", which loops both control artifacts across drive-absolute and rooted spellings; confirmed to fail with the fix reverted.

## Advisory

### A1 — the drive branch matched any `letter:` prefix, and preserved backslashes are reinterpreted downstream

Any word starting with a single letter and a colon matched, including ordinary POSIX text such as a sed `s:/old:/new:` substitution or a one-letter scp/rsync host alias, whose backslash escapes were then wrongly preserved. Separately, preserved raw text flowing into `resolveShellTarget` gets `path.win32.join`/`normalize` applied, which reinterprets those same backslashes as separators — a double-reinterpretation hazard. Reviewer could **not** construct an end-to-end false allow or false deny through the public guard from either, so did not promote it.

**Resolution**: partially fixed rather than deferred. The drive branch now rejects a second colon, so `s:/old:/new:` and similar keep normal escape processing (verified: it no longer parses as a write target). The downstream reinterpretation hazard is unchanged and remains open — recorded here rather than silently dropped.

### A2 — a doubly-damaged repository needs two archive invocations, with no signal after the first

Reviewer enumerated every code `loadV3Authority` and its callees can throw (`ARTIFACT_NAME_CONFLICT`, `ARTIFACT_NAME_MIGRATION_REQUIRED`, `NO_TASK`, `LEGACY_STATE_UNSUPPORTED`, `ORPHAN_V3_SNAPSHOT` ×2 sites, `CORRUPT_EVENT_AUTHORITY`, `MIXED_OR_INVALID_AUTHORITY` ×3 sites, plus uncoded errors from `evolveAll`/reducers/`assertV3TaskProjection` and unrecognized coded errors from `readEventStoreTail`) and traced each through `incompatibleArtifactPath`: **every one routes correctly**, with no case where a healthy artifact is archived or a real failure falls through. The residual gap: if both the event log and the snapshot are independently damaged, the log fails first, so one invocation archives only the log and the still-damaged snapshot needs a second call. No data is lost or misattributed, but the first receipt gives no hint the repository is still unreadable.

**Resolution**: not changed. Recorded as an open gap against AGENTS.md's framing of this verb as *the* way to unblock the Stop gate.
