## Blocking

### B1 ŌĆö UNC paths still evade the control-state write guard

`lib/supervision.mjs` ŌĆö `readShellValue` and `resolveShellTarget`

The Windows-path guard matched only `^[A-Za-z]:\\`, leaving UNC roots (`\\server\share`, and notably the `\\localhost\C$\...` admin-share spelling of a local drive) on the POSIX path where every backslash is eaten as an escape. Reviewer reproduced:

```
echo hi > \\localhost\C$\Users\...\.git\config
  -> targets: ["\\localhostC$Usershexin....gitconfig"]
```

The mangled target cannot resolve under `.git` or `.workloop`, so the write is not recognized as control-plane-targeted ŌĆö a false allow on a write-authorization guard, reachable because Bash-shaped tool calls default to the POSIX dialect.

**Resolution**: extracted `WINDOWS_ABSOLUTE_PATH = /^(?:[A-Za-z]:[\\/]|\\\\[^\\/])/` and used it in both places. Covered by `tests/foreign-session-scope.test.mjs` ŌĆö "a POSIX-dialect parser keeps Windows absolute write targets intact", which asserts both UNC spellings survive parsing, and also asserts a *single* leading backslash still gets ordinary POSIX escaping so the guard does not over-swallow.

### B2 ŌĆö `archive-incompatible-state` archived the healthy artifact

`lib/application.mjs` ŌĆö `cmdArchive`

The artifact choice was `fs.existsSync(snapshot) ? snapshot : eventStorePath(repo)`, a pure existence heuristic. Because `task.json` is rewritten on every successful commit, it is present and healthy in exactly the realistic corrupt-event-tail case. Reviewer reproduced: the healthy snapshot was renamed into `.workloop/archive/`, the corrupt `events.jsonl` was left untouched, and the command then threw (`exit 2`, no receipt) while recovering from the still-corrupt log. The original test passed only because it deleted `task.json` first ŌĆö engineering away the precondition that breaks the heuristic.

**Resolution**: added `incompatibleArtifactPath`, which loads the authority and dispatches on the failure code ŌĆö `LEGACY_STATE_UNSUPPORTED`, `ORPHAN_V3_SNAPSHOT`, and `MIXED_OR_INVALID_AUTHORITY` are snapshot-owned; anything else belongs to the event log. Covered by a new test, "archive-incompatible-state retires the damaged artifact, not the healthy snapshot", which keeps a healthy `task.json` in place and asserts it survives byte-identical. Confirmed to fail with the fix reverted, reproducing the reviewer's exact error.

## Advisory

**Resolution**: accepted. Added two direct tests in `tests/foreign-session-scope.test.mjs` covering drive-letter and UNC target preservation, POSIX-escape non-regression, and Windows-rule `..` traversal.

### A2 ŌĆö redundant `raw.includes("\\")` guard

**Resolution**: accepted; removed when the check moved to `WINDOWS_ABSOLUTE_PATH`.

### A3 ŌĆö `taskloop` matched as a bare substring

`install.mjs` ŌĆö `/taskloop(?:\.mjs)?\b/i` had no leading `\b`, so a wrapper script named e.g. `my-taskloop-helper.sh` would trigger the pre-rename warning. Diagnostic-only, never rewrites config, but the text would mislead.

**Resolution**: accepted; changed to `/\btaskloop(?:\.mjs)?\b/i`.

### A4 ŌĆö blast radius of archiving a whole event log (design note, not a defect)

Archiving `events.jsonl` for one bad tail record discards the task's whole history rather than a salvageable prefix. Reviewer noted this is consistent with the existing "archive it with explicit user authorization" pattern for legacy schema-2 state and flagged it only for the team to confirm the tradeoff is intended.

**Resolution**: not changed. Recorded here as an open design question.
