## Deviations

- The reviewed ten-commit sequence was split into finer green TDD checkpoints: host recipe preparation, sanitized capability fixtures, payload registry, Contract 6 projection, and artifact checkpoints landed separately. This changes only commit granularity, not phase order or activation gates.
- The installer activation check applies to every production invocation, including an explicit install home. Only the test-only `WORKLOOP_INSTALL_ISOLATED=1` switch bypasses the source-repository active-task check, so an operator cannot evade the gate by redirecting `WORKLOOP_INSTALL_HOME`.
- A Contract 6 upgrade from an installed Contract 5 release permanently pins the validated Contract 5 compatibility runtime across later pruning. A fresh Contract 6 install records a null compatibility pin because no trusted Contract 5 runtime exists to preserve.
- `task_amended` gains a Contract 6 payload-v2 form carrying `history_requirement`; Contract 5 payload-v1 bytes and replay remain frozen. This is the smallest auditable way to strengthen a running routine task from `artifact_only` to `complete` without weakening critical or finite-write invariants.
- A finite write budget now implies `history_requirement=complete`. On every currently verified non-exhaustive host, write-shaped Pre fails closed; budget counting itself is enforced again inside the pure engine so a caller cannot bypass the limit. Tests that only exercised concurrency or stdin transport no longer attach an unrelated finite-write fixture.
- A trusted tool success receipt and a trusted artifact scan are separate facts. If Post reports success but repository scanning times out, the operation remains `success` while artifact-state coverage degrades to `unknown`.
- A PostToolUse authority-commit failure first attempts a second authority transaction that persists `post-authority-commit-failure` and downgrades artifact, mutation-history, and prewrite coverage. Post stays silent with exit 0 only when that durable downgrade succeeds; if authority itself cannot accept the downgrade, Post exits 2 and reports that the completion receipt could not be persisted.
- Installer activation rollback snapshots the three stable shims, the active-release manifest, the managed-skill manifest, and every managed Claude/Codex skill tree. Versioned runtime staging remains safe retry residue; every previously visible activation byte is restored.
- Closure now binds the exact task event cursor as well as checkpoint/evidence revisions; a governance or goal amendment therefore invalidates a stale satisfied observation even when no file changed.
- Criterion-side-effect handling uses the same artifact reconciliation transaction as tool completions. The mutation, degraded unowned coverage, and side-effect observation are committed together instead of advancing an evidence revision without a checkpoint.
- Artifact checkpoints record a monotonic `captured_at_ms`; manifest identity includes entry kind (`file` or `symlink`) so a symlink/file replacement cannot collide with an unchanged content hash.
- Full mutation history is proved as a continuous, same-episode chain of exhaustive operation leases from the artifact baseline through the current checkpoint. Any artifact delta outside that chain or any episode transition immediately invalidates full mutation/prewrite coverage; an old lease can never serve as a later existence-only proof.
- Contract 6 `join`/`resume` now append an explicit event-sourced `episode-boundary` coverage interval after the lifecycle event. This intentionally advances the event/task revision twice while leaving artifact and substantive revisions unchanged; derived outcome/ledger folds also degrade on the lifecycle event itself for compatibility with earlier one-event streams.
- Every decision is preflighted through the reducer and projection validator, and production repeats that preflight with exact persisted timestamps/sequences before the irreversible event-ledger append. This closes a review-discovered path where an invalid coverage event could previously be committed before replay rejected it.
- Contract 6 operation completions now retain `tool_family` in the projection and bind it, together with `host_profile`, to the authorization. An orphan completion remains a permanent blocker to `full` even when it interleaves with an already-open exhaustive lease.
- Any artifact-changing reconciliation immediately downgrades mutation/prewrite coverage in the core, outcome projection, and ledger fold; a following valid `coverage_changed` event may restore controlled coverage within the same transaction, while an older stream that lacks that event remains honestly `unknown`.

## 2026-07-23 full-repository audit remediation

### Deviations
