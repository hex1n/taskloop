# 08 — 过夜无人值守配方

Type: grilling
Status: resolved
Blocked by: 02

## Question

维度 6（授权与安全）的裁决，owner 真实需求：睡前交付一个任务让它整夜自跑、早晨可信验收，需要什么配方？

1. **驱动侧**：选哪个驱动（/goal、codex exec、宿主 scheduled task），外层墙钟预算怎么设（spike 有 codex exec 僵死 20 分钟靠人工杀的伤疤——过夜没有人工杀兜底，外层墙钟已升为必须）。
2. **安全侧**：Claude 无头模式无沙箱旗标（已知缺口）→ 过夜跑用容器还是改走 Codex 沙箱档。
3. **预算默认**：过夜任务的 rounds/wall-clock/token 预算画像（与白天交互式不同）。
4. **挂起语义**：夜里 `needs_input` 就地停等到早晨是预期行为，确认驱动不与挂起对抗（hookStop 挂起盲已修，剩宿主侧推次浪费的确认）。
5. **早晨验收面**：起床后最少看什么就能判断「完成/卡住/需要我」——与维度 11 产出物形态衔接。

评审升格（无人值守下 weak-close 是否强制独立评审）不在本票——归票 07，此处引用其结论。

上下文：[维度框架决策记录](../../../docs/decisions/2026-07-15-loop-engineering-dimension-framework.md)；档 1 准入验证见 [taskloop vs. loop engineering](../../../docs/research/2026-07-11-taskloop-vs-loop-engineering.md) 追加 3。

## Comments

- **2026-07-15：票 07 已裁，上面那条引用的结论到了**（见[信任链决策记录](../../../docs/decisions/2026-07-15-trust-chain-authorship-review.md)「交付票 08 的结论」节）。

  **答复：不升格，因为规则本就恒定。** 评审等级由机器风险地板决定，而 07 决议 5 裁定**机器地板不可豁免**——这条**不分档位**。无人值守不是一个能豁免它的档位；`review_policy: waived` 在任何档位都只能盖住 `declared_risk` 那一半，盖不住 runtime 从自己发出的 grant、自己记的缺口算出来的事实。

  **对本票的硬约束（这是 07 给本票的真正交付物，直接约束第 1/2/3/5 问）**：过夜任务若拿 `destructive` / `install` / `whole_repo` / `publish` grant，或触发时序违规缺口（动工后 amend 判据），机器地板即 `substantial`／`critical` → 强制 `fresh_context`／`second_model` → **该任务在早晨有人评审前无法关单**。

  即：**「整夜自跑 + 早晨可信验收」自洽；「整夜自跑 + 自己关单」不自洽。** 账本里那 2 个 critical 有效风险的递归删除+重装任务零评审关单（`waiver_granted_by: self`），正是后者的样子——**而那恰恰是 owner 最不希望在无人值守时发生的事**。07 把这个逃生舱焊死了。

  故本票要设计的是**在此约束下**的配方，可能的方向（本票自裁，此处不代裁）：过夜任务的 grant 面收窄到不触发 critical 地板、把需要地板的活留给白天；或接受「早晨评审后才终态」为正常形态（这与第 5 问「早晨验收面」天然合流——早晨那一眼**就是**机器地板要求的那次评审，不是额外负担）。**不要等一个不会到来的档位豁免。**

## Answer

**票面前提被账本证否，且 07 交付的那条硬约束也是错的。** 详见[过夜配方决策记录](../../../docs/decisions/2026-07-15-overnight-unattended-recipe.md)。

- **那个夜晚没有自跑**：账本唯一跨夜任务（`1e123be8`，940m）跑了 **11 分钟**就挂在 `needs_input` 上，等了 **893 分钟零事件**——它要的输入是「人去部署测试环境」。夜里 rounds 消耗 **0**。**893 分钟是阻塞，不是算力。** 全账本 3 次挂起全属这一个任务；中位任务 21 分钟。与 06/07 同形：票面立论不是病灶。
- **约束变量重述**：不是「机器能跑多久」（问 1/2/3 全在给这个窗口做设计），而是**「机器能撑多久不需要你」**——940 分钟的夜 vs 110 分钟的最长实际工作 = **8 倍余量**，故不必赛跑、不必自愈。

**六条决议**：① 夜晚 = 我不在椅子上，不是算力窗口；② **准入式 = 判据今晚能否无人开火**（唯一死掉的夜晚死在这条上，且开单时就可判）——**08 交付的是准入规则，不是配置**；③ **档 1：交互式会话 + `/sandbox` + Stop 闸门**（`HOSTS.md:40-43`——无头才需要容器，档 1 不无头；问 1/问 2 随之出局），档 2 留雾区；④ **预算：`--rounds 30`、不设 wall-clock**（四维度里**只有 wall-clock 被睡眠烧光**——它量 `atEpochMs - createdAtMs`，`task-engine.mjs:211`）、**不设 token**（0 样本 + best effort，未经白天检验的 kill switch 不首发在夜里）；⑤ **D2「早晨评审即终态」为意图形态**（Design 1 无法预先承诺：3am 判据打嗝 → amend → 抬地板 → 随机退化成 D2），**但标注当前不可强制**；⑥ **病灶：评审闸门是 agent 自己能写的一张纸**。

**病灶（决议 6）**：`reviewer` 是自由字符串、`--blocking-findings 0` 是自填数字（`application.mjs:726`），`acceptedReview` **没有一行检查评审者是谁**（`task-engine.mjs:135-143`）。**故 07「整夜自跑 + 自己关单不自洽」是错的**——写张纸就行；C2 臂早有活证。**过夜使它变尖**：白天自签你看得见，夜里自签你早晨只看到 `achieved`。**锚点就在同一行没人用**——`acting_session`；`fresh_context` 的字面意思就是「另一个 session」。**三票同形**：06 手上有 `output_tail` 却扔了／07 手上有 tail 却没拿它分类／**08 手上有 `acting_session` 却没拿它判独立性**。

**副产品：对票 07 决议 5 的更正**——焊死 `waived`（**自报家门**：账本明写 `waiver_granted_by: self`）而不焊死 `review`（**不自报家门**：`reviewer: "…"`），等于把逃生推到一扇看不出来的门。**诚实的零评审 → 洗过的假评审**，后者更难看（账本从「没人看」变成断言「已评审、0 blocking」）。故 07 的重算「零评审 5/10 → 2/10」是**乐观的**。不推翻 07 任何一条决议，只说明焊一扇门时要看另一扇。

**问 5 由病灶反推**：早晨看 `taskloop status` 两行——`lifecycle` 行答「完成/卡住/需要我」，**`reviews` 行答「这个『完成』是不是 agent 自己签的字」**（因闸门可伪造，此行从记账升为必看）。账本掉行不要紧：权威流在仓内，`sync-outcomes --repo` 可重建。

**认下的代价**：票 12 落地前**过夜的 D2 只能靠纪律**——而这恰是最没有纪律可言的时刻。**08 想设计一个「早晨可信验收」的配方，查到最后发现：可信的那一环，runtime 现在还托不住。**

**维度 6 就此裁完**；病灶出票 [12 评审独立性锚点](12-review-independence-anchor.md)（graduate 自地图雾区「人类授权的不可伪造性」）。
